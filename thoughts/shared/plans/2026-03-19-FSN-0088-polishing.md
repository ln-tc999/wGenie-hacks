# FSN-0088: Polishing YO Treasury Hackathon Project

## Overview

Four polish items for the YO Treasury hackathon submission: fix the protocol label, add wallet connect functionality, hide deposit/withdraw on YO child vaults, and build a content-rich landing page.

## Current State Analysis

1. **Protocol label**: `plasma-vaults.json:1133` has `"protocol": "wGenie Fusion"` for the YO Treasury vault
2. **Wallet connect**: No connect button in header (`sidebar-layout.tsx:38-41`). `SidebarUser` returns `null` when not connected. Deposit/withdraw forms show "Connect Wallet" text but button is always disabled when `!userAddress`
3. **YO child vaults**: `deposit-features.tsx:28-30` renders full `YoVaultSidebar` (deposit/withdraw widget from `@yo-protocol/react`) for vaults with `yo-vault` tag
4. **Landing page**: `yo-landing-page.tsx` is minimal — just logo, headline, subtitle, 2 CTAs

### Key Discoveries:
- wagmi config (`wagmi-provider.tsx:27-31`) uses only `injected()` connector — no RainbowKit/Web3Modal
- `useConnect` + `connect({ connector: injected() })` is the pattern used in `login-content.tsx` and `transaction-proposal.tsx`
- `VAULT_TAG.YO_VAULT` constant and `hasTag()` helper already exist in `vaults-registry.ts`
- The only working inline connect button is in `transaction-proposal.tsx:332-354`

## Desired End State

1. Protocol label shows "YO Protocol & wGenie Fusion" on the treasury vault detail page
2. A "Connect Wallet" button appears in the header when no wallet is connected, triggering `injected()` connector inline
3. Deposit/withdraw form buttons are enabled and trigger wallet connection when no wallet is connected
4. YO child vault pages (`yo-vault` tag) show no deposit/withdraw sidebar
5. Landing page is a full corporate-style page selling YO Treasury to portfolio managers and hackathon judges

## What We're NOT Doing

- Not adding RainbowKit or Web3Modal — keeping the simple `injected()` connector
- Not changing the SIWE auth flow — just adding inline wallet connect
- Not adding any new data to YO child vault pages (just removing the deposit/withdraw widget)
- Not restructuring routing or page layouts

## Implementation Approach

Four independent changes that can be done in any order. Phase 4 (landing page) is the largest.

## Phase 2: Add Connect Wallet Functionality

### Overview
Add a "Connect Wallet" button to the header and make deposit/withdraw form buttons trigger connection when no wallet is connected.

### Changes Required:

#### 1. Header Connect Button
**File**: `packages/web/src/components/sidebar/sidebar-layout.tsx`
**Changes**: Add a `ConnectWalletButton` component that shows when no wallet is connected. Uses `useAccount` to check connection state and `useConnect` with `injected()` to trigger connection.

```tsx
import { useAccount, useConnect } from 'wagmi';
import { injected } from 'wagmi/connectors';
import { Wallet } from 'lucide-react';
import { Button } from '@/components/ui/button';

function ConnectWalletButton() {
  const { isConnected } = useAccount();
  const { connect, isPending } = useConnect();

  if (isConnected) return null;

  return (
    <Button
      variant="outline"
      size="sm"
      onClick={() => connect({ connector: injected() })}
      disabled={isPending}
    >
      <Wallet className="h-4 w-4 mr-2" />
      {isPending ? 'Connecting...' : 'Connect Wallet'}
    </Button>
  );
}
```

Then in the header div at line 38:
```tsx
<div className="ml-auto flex items-center gap-2">
  <ThemeToggle />
  <ConnectWalletButton />
  <SidebarUser />
</div>
```

**Note**: This component is inside `SidebarLayout` which is already `'use client'`. But it needs `AppProviders` wrapping for wagmi hooks. Check if the wagmi provider is already higher in the tree. If not, wrap `ConnectWalletButton` in `AppProviders` like `SidebarUser` does.

## Phase 4: Content-Rich Landing Page

### Overview
Rebuild the landing page (`yo-landing-page.tsx`) into a full corporate landing page that sells YO Treasury to portfolio managers, asset managers, and hackathon judges. Follow the YO design system (black bg, `#D6FF34` neon accent, Space Grotesk).

### Content Source
Derive all copy from:
- `thoughts/kuba/notes/yo-hackathon/video-voiceover-script.md` — problem statement, product description, security model, closing pitch
- `thoughts/kuba/notes/yo-hackathon/project-details.md` — features, architecture, user stories, tech stack

### Page Structure

**File**: `packages/web/src/yo-treasury/components/yo-landing-page.tsx`

The page should follow this section flow (standard corporate landing page):

1. **Hero Section**
   - YO logo with neon glow
   - Headline: "One Vault. One Deposit. AI Handles the Rest."
   - Subtitle: Problem statement — DeFi yield is fragmented
   - Primary CTA: "Open App" → demo vault
   - Secondary CTA: "Create YO Treasury" → `/yo-treasury/create`

2. **Problem → Solution Section**
   - Left: The pain (4 separate deposits, manual swaps, dozens of signatures)
   - Right: The solution (single vault, AI copilot, atomic transactions)

3. **Key Features Grid** (3-4 cards)
   - **One-Sign Batch Rebalancing** — Swap + deposit in one atomic tx
   - **AI Copilot** — Natural language → encoded calldata
   - **Fork Simulation** — See exact balance changes before signing
   - **Role-Based Security** — AI never touches keys, whitelisted actions only

4. **How It Works** (3-step flow)
   - Step 1: Deploy your vault (6-step wizard)
   - Step 2: Deposit USDC
   - Step 3: Tell the AI where to allocate

5. **Architecture / Trust Section**
   - Visual representation of the vault → fuses → YO vaults flow
   - Security highlights (no private key access, whitelisted only, simulation before every tx)

6. **Supported YO Vaults** (4 cards)
   - yoUSD, yoETH, yoBTC, yoEUR with their asset colors

7. **Tech Stack / Built With**
   - YO SDK, wGenie Fusion, Mastra, Base — brief badges/pills

8. **Final CTA Section**
   - "Start managing your treasury" + both CTA buttons again

### Design Guidelines (from yo-design skill):
- `bg-yo-black` page background, `bg-yo-dark` cards
- `text-yo-neon` for accent highlights, `text-white` for headings, `text-yo-muted` for body
- Atmospheric neon glow blurs behind key sections
- Card borders: `border border-white/5`
- `font-yo` on root div
- No gradients on brand colors
- Hover glow overlays on cards: `bg-yo-neon/[0.02]`
- Icons and numbers in neon for emphasis
- Generous negative space
- Keep existing `VAULT_URL` and `CREATE_URL` constants
