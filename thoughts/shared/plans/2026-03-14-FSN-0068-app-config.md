# App Config System Implementation Plan

## Overview

Create an app config system that lets the same Next.js web app run in different "modes" (`fusion` vs `yo`), each with its own vault list, menu items, CSS theme, feature flags, and dashboard. The config is selected at dev/build time via an environment variable `NEXT_PUBLIC_APP_CONFIG`.

## Current State Analysis

- **Vault list**: Single `plasma-vaults.json` at monorepo root with 61 vaults. Tags like `wgenie-fusion`, `yo-treasury`, `yo-vault` exist but are ad-hoc strings mixed with strategy tags (`Lending Optimizer`, etc.)
- **Conditional rendering**: Scattered `hasTag()` checks in `page.tsx:18`, `requiredTag` in `vault-tabs.config.ts:30`
- **Navigation**: Static `navItems` array in `nav-config.ts` — all 6 items always visible, YO Treasury hardcoded by address
- **Theme**: Root layout applies `class="dark"`. YO pages use `bg-yo-black font-yo` locally in `yo-treasury/layout.tsx`
- **Dashboard**: Single dashboard at `(dashboard)/page.tsx` — shows metrics across ALL vaults unfiltered
- **Vault filtering**: `ERC4626_VAULTS` used unfiltered everywhere: `fetch-vaults.ts`, `fetch-dashboard-metrics.ts`, `fetch-dashboard-rankings.ts`

### Key Discoveries:

- `vaults-registry.ts:40` — `ERC4626_VAULTS` is the single export consumed by all vault-related fetchers
- `nav-config.ts:9` — Static array with `NavItem[]`, imported directly by `app-sidebar.tsx:8` and `bottom-nav.tsx:1`
- `vault-tabs.config.ts:30` — `requiredTag: 'wgenie-fusion'` gates the Alpha tab
- `global.css:82-91` — YO brand tokens already defined (`--color-yo-neon`, `--color-yo-black`, `--font-yo`)
- `sidebar-layout.tsx:29-33` — Logo hardcoded to `logo-fusion-by-wGenie.svg`
- No `dev:web:yo` script exists — only `dev:web` in root `package.json:12`
- `yo-treasury/layout.tsx` applies `bg-yo-black font-yo` independently of the main sidebar layout

## Desired End State

- Running `pnpm dev:web:yo` starts the app in YO mode; `pnpm dev:web` (or `pnpm dev:web:fusion`) starts Fusion mode
- Each mode shows only its vaults, its menu items, its theme, and its dashboard
- `plasma-vaults.json` stays as the single source of truth but each vault has an `app` field (`"fusion"` or `"yo"`)
- Tags remain for strategy metadata — the `app` field is the strict discriminator
- Ponder continues indexing all vaults regardless of config

### Verification:

- `pnpm dev:web:yo` → sidebar shows YO nav items + YO logo, dashboard is YO-specific, vaults list shows only `yo-*` vaults, theme is `bg-yo-black font-yo`
- `pnpm dev:web` → sidebar shows Fusion nav items + Fusion logo, dashboard shows Fusion metrics, vaults list shows only `wgenie-fusion` vaults

## What We're NOT Doing

- Splitting `plasma-vaults.json` into separate files
- Changing Ponder indexing — it stays unified across all apps
- Creating separate Next.js apps or builds
- Changing Supabase schema or queries (they still query by chainId+address)
- Removing tags entirely — they remain useful for strategy metadata and tab visibility

## Implementation Approach

The config is a simple TypeScript module that reads `NEXT_PUBLIC_APP_CONFIG` and returns a typed config object. All consumers (`ERC4626_VAULTS`, nav items, theme, dashboard routing) read from this config instead of their current hardcoded sources. Minimal new files, maximal reuse.

import { getAppConfig, type NavItem } from '@/lib/app-config';

export type { NavItem };

export function getNavItems(): NavItem[] {
  return getAppConfig().navItems;
}

export function getActiveNavItem(pathname: string): string | undefined {
  const navItems = getNavItems();
  const exact = navItems.find((item) => item.url === pathname);
  if (exact) return exact.url;
  const prefix = navItems.find(
    (item) => item.url !== '/' && pathname.startsWith(item.url),
  );
  return prefix?.url;
}

import { getAppConfig, type AppConfig } from '@/lib/app-config';

interface TabConfig {
  label: string;
  description: string;
  id: string;
  featureFlag?: keyof AppConfig['features'];
}

export const TABS = [
  { id: 'overview', label: 'Overview', description: 'Key metrics and flow analysis' },
  { id: 'depositors', label: 'Depositors', description: 'Depositor information and statistics' },
  { id: 'activity', label: 'Activity', description: 'Recent transactions and activity' },
  { id: 'alpha', label: 'Alpha', description: 'Chat with AI about this vault', featureFlag: 'alphaTab' as const },
] as const satisfies TabConfig[];

export function getVisibleTabs() {
  const config = getAppConfig();
  return TABS.filter((tab) => {
    if (!('featureFlag' in tab) || !tab.featureFlag) return true;
    return config.features[tab.featureFlag];
  });
}

import { YoTreasuryOverview } from '@/yo-treasury/components/yo-treasury-overview';

  params,
}: {
