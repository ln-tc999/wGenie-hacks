# Implement activity page

- Display activity list on page http://localhost:3000/activity.
- List all deposits and withdrawals to/from Fusion Vaults.
- Show:
  - what vault is item for - with internal link to the vault
  - what kind of activity (deposit/withdraw)
  - amount
  - depositor address or ENS name with avatar - with icon links to blockexplorer and DeBank
  - transaction hash with link to blockexplorer
  - date
  
## Instructions

- Read this page https://fusion-tracker.figma.site/
  - use full browser agent
  - this is reference point
- Find right methods to gather that data
- Implement comprehensive solution to show relevant data including
  - frontend `packages/web`
  - events indexing `packages/ponder`
  - TypeScript SDK `packages/sdk`
  - tests on fork for existing smart contracts `packages/hardhat-tests`
- Remember about formating and giving context for numbers, dates, assets, addresses
- Test changes using using Browser MCP
- Prefer fetching data on frontend if possible - wagmi hooks
- Use event indexing only if you need historical, time series data
- Prefer using existing data from database, avoid indexing to blow up, keep it minimal
- Apply filtering
  - filter by:
    - specific vaults
    - depositor address
    - deposit or withdraw amount
    - date
  - filter features should be descreet - use collapsable from shadcn
- no sorting, always recent data on top
- take the best from shadcn tables to display data
- user attention should be focused on activities list
- take the best from Next.js
  - prefer server rendering for database data
  - prefer to not expose http queries on client site
- test changes in browser also for mobile
- mobile experiance should be world class
