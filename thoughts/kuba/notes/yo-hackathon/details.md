## 🚀 YO SDK Hackathon

### Build the Smartest DeFi Savings Account with YO

We're inviting developers to build the best consumer savings experience powered by the YO SDK.

YO is proven infrastructure for risk-adjusted, onchain yield. The engine already works. Now we're looking for the best UX and distribution layer to turn it into a category-defining savings product that real users trust and use every day.

---

## 🎯 Theme

Build the smartest savings account in DeFi using YO.

Examples:
- A consumer savings app
- An AI-powered yield agent
- A portfolio dashboard
- A mobile-first savings UX
- A treasury interface

**Requirement:** Integrate YO vaults via [YO SDK](https://docs.yo.xyz/integrations/technical-guides/sdk).

---

## 🛠 Technical Requirements

Projects must:
- Use `@yo-protocol/core` or `@yo-protocol/react`
- Execute real deposit and/or redeem flows
- Interact with live YO vaults on Base, Ethereum, or Arbitrum
- Demonstrate working onchain transactions

Mockups alone will not qualify.

## 📦 Submission Requirements

- GitHub repository
- 3-minute demo video
- Clear explanation of how the YO SDK was used

## 🧠 Judging Criteria

### UX Simplicity — 30%
Is this the best savings account experience in DeFi? Is it intuitive, clean, and frictionless?

### Creativity & Growth Potential — 30%
Would real users adopt this? Does it introduce thoughtful innovation? Is the user segment well defined, and does the product meaningfully solve their savings need?

### Quality of Integration — 20%
Is the YO SDK properly implemented? Are deposit/redeem flows live and correct?

### Risk & Trust — 20%
Is it safe and transparent? Are users clearly informed about what happens onchain?
